## 1. 初始化环境

### 1.1 克隆项目并初始化

```bash
git clone <repo-url>
cd RN-家居类

# 初始化 MediaCrawler submodule
git submodule update --init --recursive

# 安装 MediaCrawler 自己的依赖
cd tools/MediaCrawler && uv sync && cd ../..

# 安装项目依赖
uv sync
```

### 1.2 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，填入：

```env
MINIMAX_API_KEY=sk-cp-...            # 必填，Token Plan Key（注意格式 sk-cp-）
MINIMAX_BASE_URL=https://api.minimaxi.com/anthropic   # 文本生成接口（兼容 Anthropic SDK）
MINIMAX_VISION_MODEL=MiniMax-Text-01  # 仅作标记，图片分析走固定的 /v1/coding_plan/vlm
MINIMAX_TEXT_MODEL=MiniMax-M2.7
```

> **重要**：MiniMax 有两种 API Key 格式：
> - 按量计费 Key：`eyJhb...`（JWT 格式）
> - Token Plan Key：`sk-cp-...`
>
> 本项目图片分析使用 `/v1/coding_plan/vlm` 原生接口，**必须使用 Token Plan Key**（`sk-cp-` 开头）。两种 Key **不互通**，填错会报 401。
>
> MiniMax Token Plan 申请：https://platform.minimaxi.com/

### 1.3 初始化数据库

```bash
uv run python app/cli.py db init
```

成功后会在 `data/app.db` 创建 5 张表：`my_profile`、`items`、`reference_accounts`、`notes`、`crawl_logs`。

## 3. Step 0：初始化我的账号人设

**目的**：存储运营主体（你的账号）的人设信息，这些信息会自动注入所有笔记生成 Prompt，确保内容语气一致、不使用禁忌词。**首次必做，影响所有后续内容创作。**

### 3.1 方式一：爬虫自动推断（推荐）

提供自己的账号主页 URL，工具会自动抓取笔记数据，再用 AI 分析推断垂类定位、语气风格、话题标签等字段：

```bash
# 首次运行会弹出浏览器，用手机扫码登录小红书
uv run python app/cli.py profile init \
    --url "https://www.xiaohongshu.com/user/profile/<你的账号ID>?xsec_token=..."
```

执行后会：
1. 调用爬虫抓取你的近 30 条笔记
2. 用 MiniMax AI 分析笔记内容，推断账号人设字段
3. 打印推断结果，并写入数据库

推断完成后，补充爬虫无法自动获取的字段（如人设名、个人简介）：

```bash
uv run python app/cli.py profile edit \
    --persona-name "虾薯" \
    --persona-bio "住出租屋也要好看的家居研究员"
```

也可以在 `init --url` 的同时手动覆盖 AI 推断的字段：

```bash
uv run python app/cli.py profile init \
    --url "https://..." \
    --persona-tone "嘴硬傲娇，短句换行，先吐槽再给结论" \
    --taboos "精致,高品质,高级感"
```

### 3.2 方式二：手动填写

如果没有账号或不想爬取，直接手动填写：

```bash
uv run python app/cli.py profile init \
    --name "账号显示名" \
    --niche "家居软装/出租屋改造" \
    --persona-name "虾薯" \
    --persona-bio "住出租屋也要好看的家居研究员" \
    --persona-tone "嘴硬傲娇，短句换行，先吐槽再给结论" \
    --taboos "精致,高品质,高级感,氛围感" \
    --styles "奶油风,侘寂风" \
    --scenes "客厅,卧室,出租屋" \
    --hashtags "家居好物,软装分享,出租屋改造,装修日记"
```

### 3.3 查看当前人设

```bash
uv run python app/cli.py profile show
```

### 3.4 更新单个字段

```bash
# AI 推断结果不准确时，手动修正
uv run python app/cli.py profile edit --persona-tone "直白干脆，多用反问，先给结论再解释"
uv run python app/cli.py profile edit --taboos "精致,高品质,高级感,氛围感,质感"
uv run python app/cli.py profile edit --niche "出租屋软装改造/平价家居好物"

# 更新账号数据
uv run python app/cli.py profile edit --followers 1000
```

## 4. Step 1：录入榜样账号

**目的**：录入家居垂类做得好的账号，其风格数据会在 Step 3 生成 Prompt 时作为参考输入。

### 4.1 方式一：手动录入（推荐）

适合已经手动调研过的账号，直接填入关键数据：

```bash
uv run python app/cli.py accounts add <account_id> \
    --name "账号名" \
    --note-count 80 \
    --avg-likes 5000 \
    --avg-comments 150 \
    --avg-collects 1200 \
    --style "内容风格描述，如：走性冷淡极简路线，标题多用反问句，正文短段落" \
    --top-notes "高赞标题1,高赞标题2,高赞标题3"
```

`<account_id>` 从小红书账号主页 URL 中取：`https://www.xiaohongshu.com/user/profile/<account_id>`

### 4.2 方式二：爬虫抓取

适合需要批量获取账号笔记数据（需要浏览器扫码登录）：

```bash
# 首次运行会弹出浏览器窗口，用手机扫码登录小红书
python crawler/xhs_creator.py \
  --url "https://www.xiaohongshu.com/user/profile/<账号ID>?xsec_token=..." \
  --name "榜样账号名称" \
  --save-db
```

> 注意：`--url` 必须含 `xsec_token` 参数（从浏览器地址栏直接复制完整 URL）

### 4.3 管理榜样账号

```bash
# 列出所有账号
uv run python app/cli.py accounts list

# 查看账号详情
uv run python app/cli.py accounts show <account_id>

# 删除账号
uv run python app/cli.py accounts delete <account_id>
```

## 5. Step 2：图库管理

**目的**：建立图库，每件物品自动通过 MiniMax 分析，提取风格/材质/卖点等字段，作为后续内容创作的原材料。

### 5.1 导入单张图片

```bash
uv run python app/cli.py library add ./my-chair.jpg --title "奶油色布艺椅"
```

执行后会：
1. 将图片复制到 `assets/` 目录统一管理
2. 调用 MiniMax `/v1/coding_plan/vlm` 接口分析图片
3. 自动写入数据库（风格、材质、颜色、场景、话题标签、搭配建议、小红书卖点）

**输出示例**：
```
正在分析图片... ./my-chair.jpg
✓ 分析完成：布艺休闲椅 / 北欧简约
✓ 已添加物品 ID=1：奶油色布艺椅
  风格：北欧简约  材质：布艺+实木腿  颜色：米白
  场景：卧室/书房
  卖点：慵懒的坐感 / 治愈米白色 / 腿部实木细节
```

### 5.2 批量导入

```bash
# 逐个导入（暂无批量命令，v0.2 规划中）
for f in ./products/*.jpg; do
  uv run python app/cli.py library add "$f"
done
```

### 5.3 管理图库

```bash
# 查看所有物品
uv run python app/cli.py library list

# 按风格筛选
uv run python app/cli.py library list --style 北欧

# 查看详情
uv run python app/cli.py library show 1

# 手动补充标签
uv run python app/cli.py library tag 1 --add "性价比"
uv run python app/cli.py library tag 1 --add "出租屋友好"
```

### 5.4 跳过 MiniMax 分析

如果暂时没有 Token Plan Key，可先跳过分析直接入库：

```bash
uv run python app/cli.py library add ./my-chair.jpg --title "布艺椅" --no-analyze
# 后续在数据库中手动补充字段，或获取 Key 后重新分析
```

## 6. Step 3：生成笔记 Prompt（Skill 驱动）

**目的**：针对某件具体物品，生成完整的笔记创作 Prompt（含人设、物品分析、参考账号风格），粘贴给 Agent 生成内容。

### 6.1 生成 Prompt

```bash
# 基本用法（自动读取 my_profile 人设）
uv run python app/cli.py content draft <item_id>

# 指定参考账号风格（Prompt 中加入参考账号的内容风格描述）
uv run python app/cli.py content draft <item_id> --account-id <account_id>

# 添加额外创作要求
uv run python app/cli.py content draft <item_id> \
  --account-id <account_id> \
  --extra "这次主要打性价比角度，强调出租屋也能用" \
  --save   # --save 会在数据库创建 draft 状态的笔记记录，打印出 note_id
```

> 如果尚未执行 `profile init`，命令会给出警告并使用默认风格。建议先完成 Step 0。

### 6.2 将 Prompt 发给 Agent

将命令输出的 Prompt **完整复制粘贴到 OpenCode 对话**中。当前 session 已加载 `xiaohongshu-ops` skill，Agent 会按 skill 的 persona 和内容框架生成：

```
---标题候选（3 个）---
情绪型：买了这把椅子，出租屋也有了家的感觉
问题型：为什么我的椅子比朋友的贵一倍，但没好看一倍？
场景型：卧室角落放一把它，每天下班第一件事

---封面文案---
花 300 配出 3000 的感觉

---正文---
（第 1 段·钩子）
搬进新家的第一周，总感觉卧室差点什么...

（第 2 段·核心体验）
...

（第 3 段·搭配代入）
...

---互动引导---
你们卧室有没有这种「角落椅」？来评论区晒一下

---话题标签---
#家居好物 #软装搭配 #卧室布置 #北欧风 #布艺椅 #出租屋改造
```

> **Skill 参考**：
> - `persona.md` — 虾薯人设，傲娇嘴硬风格，短句换行，不说空话
> - `SKILL.md §4` — 通用内容模板
> - `references/xhs-home-feed-analysis.md` — 可复用结构模式

## 7. Step 4：填入内容

### 7.1 将 Agent 生成内容填入数据库

```bash

  --title "买了这把椅子，出租屋也有了家的感觉" \
  --body "搬进新家的第一周，总感觉卧室差点什么...（完整正文）" \

  --tags "家居好物,软装搭配,卧室布置,北欧风,布艺椅,出租屋改造" \
  --status ready
```

> `note_id` 是 `content draft --save` 时打印出来的 ID。如果没加 `--save`，先运行 `content list` 找到对应记录。

### 7.2 查看笔记

```bash
uv run python app/cli.py content list
uv run python app/cli.py content list --status ready
uv run python app/cli.py content show <note_id>
```

## 8. Step 5：导出与发布

### 8.1 导出 Markdown

```bash
# 打印到终端
uv run python app/cli.py content export <note_id>

# 保存到文件
uv run python app/cli.py content export <note_id> --output ./drafts/note-001.md
```

### 8.2 发布到小红书

目前发布为手动操作。后续版本（v0.2）规划接入 OpenClaw 自动化发布。

手动发布步骤：
1. 打开 [小红书创作者后台](https://creator.xiaohongshu.com/)
2. 点击「发布笔记」→「上传图文」
3. 上传 `assets/` 中对应的图片
4. 粘贴导出的标题、正文、话题标签
5. 发布前检查三要素：封面、标题（≤20字）、正文

> **Skill 参考**：`references/xhs-publish-flows.md` — 详细的图文/视频/长文发布 SOP

### 8.3 发布后更新状态

```bash
uv run python app/cli.py content edit <note_id> --status published
```

## 10. Skill 使用指南

`xiaohongshu-ops` skill 安装在 `~/.config/opencode/skills/xiaohongshu-ops/`，在 OpenCode 对话中自动加载。

### 10.1 Skill 与本项目的协作关系

| 本项目 CLI | Skill 提供 | 如何协作 |
|---|---|---|
| `content draft` | persona + 内容框架 | CLI 生成 prompt → 粘贴给 Agent → Agent 按 skill 规则输出 |
| `crawler/xhs_analyze.py` | 首页推荐流分析框架 | 爬虫输出原始数据 → 粘贴给 Agent → 按 xhs-home-feed-analysis 分析 |
| `crawler/xhs_creator.py` | 账号分析 SOP | 爬虫抓账号数据 → Agent 按 xhs-account-analysis 输出风格分析 |
| 手动发布 | 发布流程 SOP | 参考 xhs-publish-flows.md 的图文发布步骤 |

### 10.2 Skill 核心文件速查

| 文件 | 用途 |
|---|---|
| `persona.md` | 虾薯人设、语气规范、禁忌词 |
| `references/xhs-topic-ideation.md` | 选题生成 SOP、筛选标准 |
| `references/xhs-home-feed-analysis.md` | 推荐流分析框架 |
| `references/xhs-account-analysis.md` | 账号体检五维评分 |
| `references/xhs-publish-flows.md` | 图文/视频/长文发布详细步骤 |
| `references/xhs-comment-ops.md` | 评论回复 SOP、风控节奏 |
| `references/xhs-knowledge-base.md` | 知识库沉淀规范 |

### 10.3 知识库沉淀

每次完成一轮分析后，让 Agent 将结论沉淀到知识库：

```
帮我把这次选题分析结论沉淀到知识库（按 xhs-knowledge-base.md 规范）
```

知识库文件在 `~/.config/opencode/skills/xiaohongshu-ops/knowledge-base/`（本地存储，不提交 git）。

## 12. 常见问题

**Q: `library add` 报错 "Token Plan API Key 无效" 或 401？**

A: 确认 `.env` 中的 `MINIMAX_API_KEY` 是 Token Plan Key（`sk-cp-` 开头），不是按量计费 Key（JWT 格式 `eyJhb...`）。两种 Key 不互通。前往 https://platform.minimaxi.com/ 申请 Token Plan。

**Q: MiniMax 分析报错 "缺少 MINIMAX_API_KEY"？**

A: 检查 `.env` 文件是否存在且已填写 `MINIMAX_API_KEY`。注意 `.env` 在项目根目录，不是 `tools/MediaCrawler/` 下面。

**Q: `library add` 分析成功但字段为空（`—`）？**

A: MiniMax 返回的 JSON 解析异常。检查 `data/app.db` 中 `items.analysis_raw` 字段的原始内容，确认是否为有效 JSON。可用 `library tag` 手动补充标签。

**Q: `content draft` 提示"尚未初始化账号人设"？**

A: 先执行 `profile init` 初始化人设（见 Step 0）。命令会在没有人设时使用默认风格并给出警告，但建议填写真实人设以确保笔记语气一致。

**Q: `content draft` 的 Prompt 风格不对，如何调整？**

A: 通过 `profile edit --persona-tone "..."` 更新语气描述，或 `profile edit --taboos "词1,词2"` 更新禁忌词。这些变更会立即反映在下一次 `content draft` 的输出中。

**Q: 爬虫运行后没有数据，或者提示需要登录？**

A: 首次运行会弹出浏览器，需要用小红书 App 扫码登录。登录成功后会在 `tools/MediaCrawler/browser_data/` 缓存登录态，下次不用重复登录。如果缓存失效，重新扫码即可。

**Q: 爬虫脚本在 `crawler/` 目录下运行报错？**

A: 必须从项目根目录运行：
```bash
# 正确
cd /path/to/RN-家居类
python crawler/xhs_search.py ...

# 错误
cd crawler && python xhs_search.py ...
```
